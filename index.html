candor

The perfect start for your new project.

candor is a CSS design system which shines especially by its simplicity.

Candor is the ideal head start to your web project. It has nearly all exisiting HTML elements included, structured and styled in a way so it can easily changed. The mobile first approach gives you the possibile to design easily for smartphones, tables and desktop.

The gulp workflow automation tool brings gives you an even more efficient developement enviroment, so you can built cool stuff even faster!

All components are easy to find in the strucutred scss, that automatically compile to a minified style.css. Also, javascript files are automatically concated and minified. jQuery runs still on 73% of all websites on the web and will also on yours with candor if you don‚Äôt remove it. This gives you super powers when you like to design a Wordpress theme, because candor is already built with Wordpress theme development in mind.

Finally, accessibility is a main aspect of the design and development of candor, so everyone can enjoy your websites.

Gulp automation

Gulp is a toolkit to automate & enhance your workflow. Repetitive tasks can be automated, so you can hyper focus on the development. Luckily candor has a gulpfile already included, so you do not even have to set up the environment.

Stylesheet
The gulp automation tool prefixes the stylesheet for all modern browsers, compiles your scss files to style.css, minifies the style.css to style.min.css, adds a sourcemap.
Javascript
The gulp automation tool has plumper included, so you easily see if the code has errors. It concacts all files in ‚Äúassets/js/src‚Äù, compress and minifies them to script.js
Images
All images in from ‚Äúassets/img/src‚Äù will be minified and optimized for web saved to ‚Äúassets/img‚Äù. The source files will not be touched
Browser Sync
Browser Sync is a great tool to test your website on multiple devices simoustanly. More info.
Release task
Ready to release your website? The release task automatically compress the files and packs than into a zip in the ‚Äú_release‚Äù folder.

To run the gulp tasks, navigate in the terminal to your project folder and run the default gulp task with the gulp command while your server is up.
You might configure the gulpfile for your environment.

All gulp tasks

gulp img-min      // minifies images from ./assets/img_raw to ./assets/img
gulp sass         // compiles files from ./assets/scss/*.scss to style.css
                      * autoprefixer included
                      * sourcemaps included
gulp css-min      // minifies style.css to style.min.css
gulp js           // concats all files from ./assets/js/src to ./assets/js/script.js
gulp release      // packages the theme to a zip file in the _release folder. Takes the version number from package.json

gulp watch        // has an eye on the scss and js files
gulp min          // same as default but additionally with css-min and img-min included
gulp              // is the default task, starts browser sync, runs the js, sass the watch task

Helpers

breakpoints

assets/scss/02_helpers/breakpoint.css since v.1.0

candor uses the mobile first approach. Everything is optimized for the smartphone by default, unless otherwise specified by breakpoints.

Three breakpoints can used with the ‚Äúbp‚Äù mixin
  1. default, for devices (e.g. smartphones) smaller than 768px. The breakpoint should only be used if the style should only been display on small devices but not the others.
  2. medium, for devices (e.g. tables) between 768px and 1024px
  3. large, for devices (e.g. desktop PCs) larger than 1024px
/* Only for small devices */
@include bp(small){ YOUR CSS; }

/* For medium devices */
@include bp(medium) { YOUR CSS;}

/* For large devices */
@include bp(large) { YOUR CSS;}
In the .site-footer.scss is a little dev tool that adds a top border to the footer and adds a visualiation to the breakpoint.‚Ä®Simply uncomment the code. Don‚Äôt forget to comment it in again when releasing your design.‚Ä®pink for small display devices, powderblue for medium, palegreen for large.

pixel 2 em

assets/scss/02_helpers/pixel2em.css since v.1.0

Using em in CSS has its adventages but it also sometimes makes it difficult to keep consitency. For all who want to program with pixels, but don't want to miss the advantages of ems, this mixin is optimal. It automatically renders pixel to ems. It can also be used for rem. 16 is default.

Read more: https://css-tricks.com/snippets/sass/px-to-em-functions/ 

/* Usage: */
margin: em(32);

/* Output: */
margin: 2em;
  /* Usage: */
  margin: rem(32);

  /* Output depending on base font-size. Usually with 16px:*/
  margin: 2em;
